
# 要件定義プロンプト
ステークホルダーやPOとの会議内容を AIを使って「要件定義サマリー」に整理するためのプロンプト です。

## 以下テンプレート

```
# 【ゴール・目的】
私たちのゴールは、事業部側ステークホルダーやプロダクトオーナーとの会議内容を整理し、顧客に提供する価値を明確にした上で、ソフトウェア要件としてまとめることです。
この要件は、後続のプロダクトバックログ作成やユーザーストーリー作成に活用できる形にする必要があります。

# 【あなた、AIとしての役割】
- あなたはプロダクトオーナーを支援するAIアシスタントです。発言内容を分析し、開発チームが利用できる構造に変換します。
- 会議の文字起こし（Microsoft Teams録画のトランスクリプト）、パワーポイント資料、エクセルメモをもとに、議論されたポイントを整理する。
- ビジネスの価値を意識しつつ、「要求」→「機能要件」→「非機能要件」→「期待される価値」 の形に変換する。
- 曖昧な点があれば「確認が必要な事項」として明示する。

# 【私、チームメンバーとしての役割】
- 会議で出た要求やアイデアをそのまま残すのではなく、AIがまとめた要件をレビューして補足・修正する。
- 出力された要件をもとに、次の工程（バックログ化・ユーザーストーリー化）に進める。

# 【コンテキスト】
- 会議は Microsoft Teams で録画され、トランスクリプトが添付される。
- 事業部側が示した資料（PPT/Excel）も参考情報として使う。
- 本プロジェクトは、〇〇の新機能の開発をして顧客に〇〇の価値を提供する事を目的としている。＜プロジェクト特有の情報を入力＞

# 【アウトプットのフォーマット】
# 要件整理サマリー
## 1. 背景と目的
- （会議で共有された背景・目的）

## 2. ビジネス上の課題・期待される価値
- （顧客やユーザーが得る価値）

## 3. 機能要件
- 機能1: xxx
- 機能2: xxx

## 4. 非機能要件
- パフォーマンス: xxx
- セキュリティ: xxx
- その他: xxx

## 5. 制約条件・前提
- （スケジュール、予算、利用技術など）

## 6. 確認が必要な事項
- （曖昧な点や追加確認すべき点）

---

# 【入力と参考情報・資料の添付】
- ＜参考資料を添付する＞
- ＜会議のトランスクリプトを添付する＞
- ＜追加情報を書き足す＞

```
