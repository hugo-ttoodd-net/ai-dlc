# バックログ作成プロンプト
整理した要件定義サマリーを基に、プロダクトバックログを生成するためのプロンプトです。

## 以下テンプレート

```
# 【ゴール・目的】
私たちのゴールは、要件定義サマリーをもとに、開発すべき機能を「エピック → フィーチャー → ユーザーストーリー」へ分解し、プロダクトバックログとして整理することです。
バックログは、顧客価値やビジネス目的が分かる形に構造化し、今後のユーザーストーリー作成やスプリント計画に活用できるようにします。

# 【あなた、AIとしての役割】
- あなたは プロダクトオーナーを支援するAIアシスタントです。
- 「要件定義サマリー」およびチームから提供される技術的実装メモを分析し、プロダクトバックログを出力する。
- ビジネス価値・顧客へのメリットを明確化しながら、粒度の適切なユーザーストーリーに落とし込む。
- 不明確な部分は「追加検討が必要」と明示する。

# 【私、チームメンバーとしての役割】
- AIが生成したバックログをレビューし、技術的制約・実装方法・詳細仕様 を追記する。
- 出力されたバックログをそのまま次のステップ（ユーザーストーリー定義・スプリント計画）に渡せる状態にする。
- 事業部への確認が必要な事項があれば、明示的にリストアップしてPOと確認する。

#【コンテキスト】
- 「要件定義サマリー」（会議記録・資料を整理したもの）が入力として提供される。
- チームの技術メンバーが「実装の観点からの補足情報（利用するフレームワーク、DB設計上の前提、外部サービス連携など）」を入力する。
- 本プロジェクトは、顧客に〇〇の価値を提供する新機能開発を目的としている。＜プロジェクト特有の情報を入力＞

# 【アウトプットのフォーマット】
以下の表形式で出力してください。

| エピック | フィーチャー | ユーザーストーリー | 顧客へのメリット | 目的（ビジネス価値） | Acceptance Criteria（受け入れ条件） |
| ---- | ------ | --------- | -------- | ---------- | --------------------------- |
|      |        |           |          |            |                             |
|      |        |           |          |            |                             |


# 【入力と参考情報・資料の添付】
- ＜要件定義サマリー（前段階でAIが作成したもの）＞
- ＜Microsoft Teams 会議トランスクリプト＞
- ＜ステークホルダー提供資料（PPT/Excel）＞
- ＜技術メンバーの補足メモ（実装に関する前提・注意事項など）＞
```
